## transformToReactTree 重新实现方案

### 核心设计

将 `transformToReactTree` 改为**工厂函数**，每个 surfaceId 对应一个独立实例，各自维护组件缓存。

### 数据结构

**ReactComponentTree（产物）**：

```ts
interface ReactComponentTree {
  type: string;
  props: Record<string, any>; // 不含 id、component、children、child、value 等内部字段
  children?: ReactComponentTree[]; // 保留 id 引用：{ type: '__placeholder__', props: { id: string } }
}
```

### 两个版本的数据差异

| 字段       | v0.8                                   | v0.9                |
| ---------- | -------------------------------------- | ------------------- |
| 组件类型   | `component: { [type]: { ...config } }` | `component: string` |
| value      | `PathValue \| LiteralStringValue`      | `PathValue`         |
| 其他 props | 在 config 对象内                       | 直接在根对象上      |

### 产物统一格式

两个版本最终产出完全一致的 `ReactComponentTree`：

- `type`：组件类型字符串
- `props`：组件属性（排除 id、component、children、child、value 等内部字段；value 统一处理为字符串）
- `children`：子节点数组，保留 id 引用 `{ type: '__placeholder__', props: { id: string } }`

### 实现步骤

1. **修改 `components.ts`**：
   - 定义 `ReactComponentTree` 接口（保持不变）
   - 新增 `createComponentTransformer()` 工厂函数，返回一个带状态的 transformer 实例
   - 实例内部维护 `componentMap: Map<string, ReactComponentTree>`（id → 节点）
   - 实例暴露 `transform(components, version)` 方法：
     1. 将新增/更新的 components 解析后合并进 `componentMap`
     2. 若 `componentMap` 中存在 `id === 'root'` 的节点，则返回该节点（children 保留 id 引用）
     3. 否则返回 `null`
   - 保留原有 `transformToReactTree` 函数签名作为默认导出（向后兼容），但内部改为使用单例工厂

2. **v0.8 节点解析逻辑**：

   ```
   const [type, config] = Object.entries(component.component)[0]
   props = 排除 child/children/value 的 config 字段
   value = config.value?.path ?? config.value?.literalString
   children = config.children ?? (config.child ? [config.child] : [])
   ```

3. **v0.9 节点解析逻辑**：

   ```
   type = component.component
   props = 排除 id/component/child/children/value 的字段
   value = component.value?.path
   children = component.children ?? (component.child ? [component.child] : [])
   ```

4. **统一 value 处理**：若 value 存在则加入 props，否则不加

5. **children 保留 id 引用**：

   ```ts
   children: childIds.map((id) => ({ type: '__placeholder__', props: { id } }));
   ```

6. **更新 `Card.tsx`**：
   - 使用 `createComponentTransformer()` 创建实例（在组件 mount 时创建，或用 `useRef` 保持引用）
   - 每次 commands 变化时调用 `transformer.transform(components, version)`

### 文件改动范围

- `/packages/x-card/src/A2UI/format/components.ts`：主要改动，重写工厂函数
- `/packages/x-card/src/A2UI/Card.tsx`：使用新的工厂函数 API
